2. ENHANCED EXAMPLE (The "Tall Person" Analogy):
------------------------------------------------
In normal (crisp) logic, a person is either "Tall" (1) or "Not Tall" (0). If the limit is 6 feet:
- A person 5'11" is Not Tall (0).
- A person 6'0" is Tall (1).
This is very strict and unrealistic!

In FUZZY LOGIC, we use degrees of truth:
- A person 5'0" is Tall with degree 0.0.
- A person 5'11" is Tall with degree 0.8 (Almost tall).
- A person 6'5" is Tall with degree 1.0 (Definitely tall).
Fuzzy logic allows us to mathematically handle words like "Somewhat", "Very", or "Fairly".

3. SIMPLIFIED CONCEPT EXPLANATION:
----------------------------------
Fuzzy logic handles the "gray area" between absolute True and absolute False. 

BASIC OPERATIONS:
- UNION (OR): If I am "Kind" (0.8) OR "Rich" (0.4), how happy am I? We take the MAX value (0.8).
- INTERSECTION (AND): If I am "Kind" (0.8) AND "Rich" (0.4), how likely am I to donate? We take the MIN value (0.4).
- COMPLEMENT (NOT): If I am "Happy" (0.7), then I am "Not Happy" (1 - 0.7 = 0.3).

FUZZY RELATIONS:
A relation describes how elements of two sets are connected (e.g., how "Humidity" relates to "Rain").
- Cartesian Product: Finds the relationship strength for every possible pair (x, y).
- Max-Min Composition: Chains two relations together. If X relates to Y, and Y relates to Z, what is the direct relation between X and Z?

4. DETAILED CODE SNIPPET EXPLANATION (For Examiner):

[CODE]
def get_fuzzy_set(name):
    s = {}
    n = int(input(f"How many elements in Fuzzy Set {name}? "))
    for _ in range(n):
        elem = input(f"  Element name: ")
        deg  = float(input(f"  Membership degree (0-1): "))
        s[elem] = deg
    return s
[EXPLANATION]
This function builds a "Fuzzy Set" by asking the user for inputs. It stores them in a Python dictionary where the key is the element name and the value is the membership degree.

[CODE]
def fuzzy_union(A, B):
    keys = set(A) | set(B)
    return {k: max(A.get(k, 0), B.get(k, 0)) for k in sorted(keys)}

def fuzzy_intersection(A, B):
    keys = set(A) | set(B)
    return {k: min(A.get(k, 0), B.get(k, 0)) for k in sorted(keys)}
[EXPLANATION]
1. Union (OR): For every element `k` in either set, it calculates `max(A[k], B[k])`. If an element is missing in one set, it uses 0 as its degree.
2. Intersection (AND): For every element, it calculates `min(A[k], B[k])`. This represents elements that are "members of both sets".

[CODE]
def fuzzy_complement(A):
    return {k: round(1 - v, 4) for k, v in A.items()}

def fuzzy_difference(A, B):
    keys = set(A) | set(B)
    return {k: round(min(A.get(k, 0), 1 - B.get(k, 0)), 4) for k in sorted(keys)}
[EXPLANATION]
1. Complement (NOT): Calculates `1 - membership_degree`. If you are 0.8 "Happy", you are 0.2 "Not Happy".
2. Difference (A - B): Defined as the intersection of A and the complement of B. It calculates `min(degree_A, 1 - degree_B)`.

[CODE]
def cartesian_product(A, B):
    return {(x, y): round(min(ux, uy), 4) for x, ux in A.items() for y, uy in B.items()}
[EXPLANATION]
This creates a matrix of relations. For every element `x` from set A and every element `y` from set B, it calculates the relationship strength using `min(membership_x, membership_y)`.

[CODE]
def max_min_composition(R, S, A_keys, B_keys, C_keys):
    return {
        (x, z): round(max(min(R[(x, y)], S[(y, z)]) for y in B_keys), 4)
        for x in A_keys for z in C_keys
    }
[EXPLANATION]
This is the most complex part. To find the relation between `x` and `z`:
1. It looks at all possible intermediate connections `y`.
2. `min(...)`: Finds the "weakest link" strength for the path `x -> y -> z`.
3. `max(...)`: Picks the "strongest" overall path among all available intermediates `y`.

5. VIVA QUESTIONS AND ANSWERS:
------------------------------
Q1: What is the range of a membership degree?
A: It is always between 0.0 (not a member) and 1.0 (fully a member).

Q2: How does Fuzzy Union differ from Crisp Union?
A: In Crisp sets, union is "either/or". In Fuzzy sets, union is the "maximum" of the membership values of the elements.

Q3: What is a "Linguistic Variable"?
A: A variable whose values are words instead of numbers, like "Temperature" being "Hot", "Warm", or "Cold".

Q4: Where is Fuzzy Logic used in real life?
A: Washing machines (to sense dirt), ACs (to adjust cooling), and Anti-lock Braking Systems (ABS) in cars.

Q5: What is a membership function?
A: A mathematical curve that defines how each point in the input space is mapped to a membership value between 0 and 1.
